# M6 — Model Card Generation (Implementation Plan)

## Context

M5 delivered the end-to-end tabular loop, but the Results screen shows only a placeholder model card. M6 auto-generates the doctor-facing `model_card.md` from the experiment data, using the spec template.

M6 is **contract-critical**: the model card is the primary artifact doctors see. It must be readable, accurate, and include all safety disclaimers. The card is written to `~/.doclab/experiments/<id>/model_card.md` and the path is stored in the SQLite `experiments.model_card_path` column.

## Goal

Auto-generate a doctor-facing markdown model card for every completed experiment. The card includes: goal, intended use, summary (dataset/task/model/result), limitations, risks, reproducibility details, and the non-clinical disclaimer.

## Spec Template (from spec.md:241-270)

```markdown
# <Model Name>

**Goal:** <original doctor text>
**Intended use:** Research and prototyping only—not for clinical decisions.

## Summary
- **Dataset:** <name> (public / de-identified)
- **Task:** <e.g. binary classification>
- **Model:** <e.g. XGBoost classifier>
- **Result:** <e.g. 83% accuracy> (baseline: <X>% for tabular)

## Limitations
- Trained on a public prototype dataset; may not generalize to real hospitals.
- <Dataset-specific limitation>

## Risks
- Not validated for patient care. Do not use for diagnosis or treatment decisions.

## Reproducibility
- Dataset: `<hf_id>@<revision>`
- Seed, split ratio, hyperparameters: see `plan.json`

## Examples (text runs only)
<3 input/output pairs>
```

## Implementation Approach

**Where to generate:** Rust, after worker succeeds. The model card is written immediately after `metrics.json` is parsed, before updating the SQLite row to `complete`.

**Why Rust, not frontend:** The card is an artifact of the experiment, not a UI concern. It should exist on disk even if the user never opens the Results screen. Rust owns the experiment lifecycle and has all the data (plan, metrics, dataset metadata) in one place.

### New Rust Function

**`src-tauri/src/experiments.rs`**:
```rust
fn generate_model_card(
    experiment_dir: &Path,
    goal_text: &str,
    dataset: &Dataset,
    plan: &WorkerPlan,
    metrics: &MetricsBlob,
) -> Result<String, String> {
    let model_card_path = experiment_dir.join("model_card.md");
    
    // Model name: sanitize goal text (first 50 chars, alphanumeric + spaces)
    let model_name = sanitize_model_name(goal_text);
    
    // Task type: infer from plan or default to "classification"
    let task = infer_task_type(plan);
    
    // Model display name: map model_type to lay language
    let model_display = match metrics.model_type.as_str() {
        "xgboost" => "Gradient-boosted decision trees (XGBoost)",
        "logistic_regression" => "Logistic regression",
        _ => &metrics.model_type,
    };
    
    // Result formatting
    let result_pct = (metrics.metric_value * 100.0).round() as i32;
    let baseline_pct = (metrics.baseline_metric * 100.0).round() as i32;
    
    // Sanity check message
    let sanity_note = if metrics.metric_value <= metrics.baseline_metric + 0.02 {
        "\n\n**Note:** Model performance is close to the baseline. The model may not be learning meaningful signal from the data."
    } else {
        ""
    };
    
    // Dataset-specific limitation (from marketplace entry)
    let dataset_limitation = if !dataset.limitations.is_empty() {
        format!("- {}", dataset.limitations)
    } else {
        "".to_string()
    };
    
    let card = format!(
        r#"# {model_name}

**Goal:** {goal_text}
**Intended use:** Research and prototyping only—not for clinical decisions.

## Summary
- **Dataset:** {dataset_name} (public / de-identified)
- **Task:** {task}
- **Model:** {model_display}
- **Result:** {result_pct}% accuracy (baseline: {baseline_pct}%){sanity_note}

## Limitations
- Trained on a public prototype dataset; may not generalize to real hospitals.
{dataset_limitation}

## Risks
- Not validated for patient care. Do not use for diagnosis or treatment decisions.
- False positives or false negatives may lead to inappropriate clinical actions.
- Model performance may degrade on populations not represented in the training data.

## Reproducibility
- **Dataset:** `{hf_id}@{revision}`
- **Seed:** {seed}
- **Split ratio:** {split_display}
- **Device:** {device}
- **Framework:** {framework}
- **Hyperparameters:** See `plan.json` in the experiment directory.

---

*This model card was auto-generated by DocLab. For research and prototyping use only.*
"#,
        model_name = model_name,
        goal_text = goal_text,
        dataset_name = dataset.name,
        task = task,
        model_display = model_display,
        result_pct = result_pct,
        baseline_pct = baseline_pct,
        sanity_note = sanity_note,
        dataset_limitation = dataset_limitation,
        hf_id = plan.hf_id,
        revision = &plan.revision[..7.min(plan.revision.len())],
        seed = plan.seed,
        split_display = format!("{:.0}/{:.0}/{:.0}", plan.split[0]*100.0, plan.split[1]*100.0, plan.split[2]*100.0),
        device = metrics.device,
        framework = metrics.framework,
    );
    
    fs::write(&model_card_path, card)
        .map_err(|e| format!("cannot write model_card.md: {e}"))?;
    
    Ok(path_string(&model_card_path))
}

fn sanitize_model_name(goal: &str) -> String {
    let clean: String = goal
        .chars()
        .take(50)
        .filter(|c| c.is_alphanumeric() || c.is_whitespace())
        .collect();
    if clean.trim().is_empty() {
        "Prototype Model".to_string()
    } else {
        clean.trim().to_string()
    }
}

fn infer_task_type(plan: &WorkerPlan) -> &str {
    // Phase 1: all tabular runs are binary classification
    "Binary classification"
}
```

```rust
fn update_complete(
    conn: &Connection,
    id: &str,
    metrics: &MetricsBlob,
    metrics_path: &str,
    model_card_path: &str,  // NEW
    stdout: &str,
    stderr: &str,
) -> Result<(), String> {
    conn.execute(
        "UPDATE experiments SET
            updated_at_ms=?2, status='complete', primary_metric=?3,
            metric_value=?4, baseline_metric=?5, model_type=?6, framework=?7,
            device=?8, metrics_path=?9, model_card_path=?10, worker_stdout=?11, worker_stderr=?12,
            error_code=NULL, error_message=NULL
         WHERE id=?1",
        params![
            id,
            now_ms(),
            metrics.primary_metric,
            metrics.metric_value,
            metrics.baseline_metric,
            metrics.model_type,
            metrics.framework,
            metrics.device,
            metrics_path,
            model_card_path,  // NEW
            stdout,
            stderr,
        ],
    )
    .map(|_| ())
    .map_err(|e| format!("cannot update completed experiment '{id}': {e}"))
}
```

## Return Card Content Server-Side (No fs Plugin)

**Decision:** Return the rendered card text in `ExperimentDetail` rather than reading the file from the frontend.

**Why:** `get_experiment_by_id` already reads `metrics.json` from disk and returns it as a parsed `Value` in `ExperimentDetail.metrics`. The model card should follow the same pattern — read the file server-side and return its text. This avoids adding `tauri-plugin-fs` (and its scope/permission config), keeps the frontend dumb, and is consistent with how artifacts already flow to the UI.

**Add field to `ExperimentDetail`** (`src-tauri/src/experiments.rs`):
```rust
pub struct ExperimentDetail {
    // ... existing fields ...
    pub model_card_path: Option<String>,
    pub model_card_content: Option<String>,  // NEW
    // ...
}
```

**Populate in `get_experiment_by_id`** — after reading `model_card_path` from the row, read the file if present:
```rust
let model_card_content = model_card_path
    .as_ref()
    .and_then(|p| fs::read_to_string(p).ok());
```
Mirror the existing pattern used for `metrics` (which is read from `metrics_path` and parsed). Set `model_card_content: None` on the running/failed paths where no card exists.

**Frontend type** (`src/types/tauri.ts`) — add to `ExperimentDetail`:
```ts
modelCardContent: string | null;
```

```rust
fn model_card_generation() {
    let temp_dir = std::env::temp_dir().join(format!("doclab_m6_{}", generate_experiment_id()));
```
